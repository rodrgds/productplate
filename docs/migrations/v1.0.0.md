# Product Plate v1.0 migration

- Generate a fresh app for the selected profile and compare `product-plate.json` before adopting profile changes.
- Add separate preview and production Convex deploy keys before enabling the unified workflow.
- Set `AUTH_REQUIRE_EMAIL_VERIFICATION=true`, Resend delivery, final legal text, support email, PostHog consent, and Sentry DSN before strict launch verification.
- Run `bunx product-plate upgrade --check`. The upgrade command changes only unchanged files listed in `.product-plate/managed-files.json`.
- Apply product schema changes manually with widen-migrate-narrow. The upgrade command never overwrites schemas or business routes.
